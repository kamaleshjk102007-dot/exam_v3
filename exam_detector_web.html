Capabilities

Built For
Real Exams

Every component is designed for exam hall conditions — crowded frames, low lighting, wide-angle CCTV, and long monitoring sessions.

🎯
Per-Class Confidence

Each behavior class has its own confidence threshold tuned to its detection difficulty. Phone requires 0.70, subtle glances only 0.45.

Precision Tuning
📸
Auto Screenshots

Violation frames are automatically saved with timestamp and class name. Independent cooldown timer prevents screenshot flooding.

Evidence Capture
🔔
Audio Alarm

880Hz stereo beep fires on sustained violations. Runs on a daemon thread so it never blocks the detection loop or drops frames.

Non-Blocking
📊
Live HUD Overlay

Semi-transparent overlay shows real-time FPS, violation counter, timestamp, and cumulative detection counts per class.

Operator View
🚫
Class Ignore System

Any class can be fully disabled with one line — no box, no alarm, no count. Stand Up and Normal ignored by default to eliminate false positives.

Zero False Positives
🔲
Box Size Filtering

Min and max area ratios reject tiny background noise and oversized false detections. Phone also gets an aspect-ratio check for portrait orientation.

Pipeline

How It
Works

01
Frame Capture

Reads frames from webcam or video file. Rolling 30-frame buffer calculates real-time FPS with no stutter.

02
YOLOv8 Inference

Each frame runs through YOLOv8 weights trained on 7 exam behavior classes. All ignored classes are skipped immediately.

03
Filter & Validate

Detections pass through 5 filters: confidence threshold, degenerate box, min area, max area, and aspect ratio for phones.

04
Alarm & Save

Alert classes trigger audio beep on daemon thread, red border flash, and auto-screenshot — all with independent cooldown timers.

Quick Start

Up & Running
In Seconds

One command is all it takes. Point it at a video file or webcam and monitoring begins immediately.

KEYBOARD CONTROLS
S — Manual Screenshot
Q / ESC — Quit
PowerShell — exam_detector_v3.py
# Install dependencies
PS X:\> pip install ultralytics opencv-python pygame numpy

# Run on CCTV video file
PS X:\> python exam_detector_v3.py --source cctv.mp4

# Run on webcam
PS X:\> python exam_detector_v3.py --source 0

# Custom model + override confidence
PS X:\> python exam_detector_v3.py --model best.pt --conf 0.55
